# 1) SIMPLE MODE
#######################################################################

=======================================================================
1.1 What this contract does (plain words)
=======================================================================
This contract runs auctions for Cosmic Mad Scientist NFTs.

- People bid using Standard Mad Scientist NFTs.
- Highest qualifying bid wins the Cosmic.
- Winner's Standard NFTs go into a pool.
- People can later swap with that pool 1:1.

Think of it like:
- "Cosmic" = prize
- "Standard Mad" = bidding chips
- "Pool" = community inventory of chips from winners


=======================================================================
1.2 Who can do what
=======================================================================
Normal users can:
- Bid on auctions (with Standard Mad NFTs)
- Withdraw losing bids
- Swap with pool

Admin can:
- Create Cosmic auctions (by depositing Cosmic)
- Pause/unpause deposits
- Update config limits
- Force complete stuck finalizing auction
- Transfer admin role (two-step)


=======================================================================
1.3 The two most important trust checks
=======================================================================
A) No double-dipping
- Same Standard Mad NFT cannot be used to bid in two live auctions.

B) No winner / bad case safety
- If no valid winner exists, Cosmic goes back to original depositor.
- Not "whoever is admin now", but who originally deposited Cosmic.


=======================================================================
1.4 What happens in each outcome
=======================================================================
Case A: Auction has winner
- Winner gets Cosmic.
- Winner's Standard Mad NFTs go into pool.
- Losers withdraw their own NFTs back.

Case B: Auction ends with no qualifying winner
- Cosmic returns to original depositor.
- Bidders can still withdraw escrowed NFTs.

Case C: Admin cancels before bids
- Cosmic returns to original depositor.


=======================================================================
1.5 What can go wrong and how contract responds
=======================================================================
- Wrong collection sends NFT -> rejected.
- Bid before start or after end -> rejected.
- Trying to reuse a bid NFT across auctions -> rejected.
- Trying to cancel after bidders joined -> rejected.
- Sending ATOM/native funds with execute msg -> rejected.
- Contract paused -> new deposits/bids rejected.


=======================================================================
1.6 One-screen "is this sound?" checklist
=======================================================================
[ ] Same bid NFT cannot appear in two live auctions.
[ ] Winner gets Cosmic only after finalize and after end time.
[ ] No winner returns Cosmic to original depositor.
[ ] Losers can withdraw in Finalizing/Completed.
[ ] Cancel only works before any bidder joined.
[ ] Swap is strictly 1:1 (deposit N, claim N).
[ ] Admin controls are restricted and reversible (two-step transfer).
